# 리엑트 강의를 들어가며...
다른 ES6 및 웹팩 보다는 리엑트에 집중하고, 리덕스 및 리덕스 사가를 통해 상태관리 및 비동기 통신에 대한 실습을 하며 공부를 진행하고 프로젝트를 통해 복습을 진행합니다.
## 리엑트는 뭘까요??!
페이스북에서 개발하고 관리하는 UI라이브러리이며 앵귤러와 달리 단지 리엑트는 UI기능 만을 관리합니다. 따라서 개발자가 개발환경 STORE, ROUTER 등을 직접 설정을 해야합니다. 그래서 Create-react-app을 통해서 어느 정도 해결을 했다고 합니다.
### 그렇다면 리엑트를 왜 사용할까요??!
이유 중 하나는 UI를 자동으로 업데이트해 주기 떄문입니다. 이벤트 또는 통신을 상태값이 변화가 되면 리엑트가 자동으로 상태를 업데이트 해줍니다.
#### TIP!
1. render() 함수 내부에는 순수함수(입력값 같으면 출력값 같음)여야합니다. 따라서 render함수안에서 상태가 변화하거나 Date 함수를 직접 넣어서 사용하는 것을 자제합니다. 왜냐하면 입력값이 같음에도 값이 달라질 수 있고 상태가 변경되면 다른 render함수에 영향을 미치기 때운입니다.
2. 상태는 불변변수로 관리하는게 좋습니다.
- 이렇게 하면 rendering 성능 및 코드의 가독성에 좋습니다.
### 가상돔
react는 가상돔을 통해서 UI를 빠르게 업데이트합니다. 이전 UI를 메모리에 저장되어 있고 변경된 부분만 실제돔에 반영하는 기술로 성능 향상에 도움이 됩니다. 실제 돔을 해석하고 다시 그리는 일은 매우 무거운 작업이라고 합니다. 이 해석을 최소화하기 위해 변경된 부분만을 해석하고 다시 랜더링 할 수 있도록하는 기술이 가상 돔입니다. 하지만 프로젝트에 따라 효율성이 다를 수 있다고 합니다.
# 개발 환경 구축하기
본격적으로 이전의 리엑트로 사고하기에 이어 리엑트에 대한 기본 개념 및 사용 이유에 대해 알아 보았습니다.
### 간단히 create-react-app으로 시작해보기~!
리엑트 개발환경을 직접 만드는 것은 조금 힘듭니다 webpack, babel등을 통해 빌드 시스템을 구축하고 테스트 환경을 위해서 jest 등을 성절할 수 있어야합니다.
> npx create-react-app my-app

위 명령어를 통해서 앱을 만들어 봅겠습니다.
리엑트 개발환경을 구축해줍니다. 하지만 빌드시스템 및 eslint 등의 설정을 변경하는데 어려움이 있습니다. 즉, 쉽고 빠르게 커스터마이징 없이 개발할 경우 아주 적합합니다.

#### 추가적으로 코드를 살펴보며 간단한 TIP들이 있습니다.

~~~
import logo from './logo.svg';
import './App.css';

function App() {
  return (
    <div className="App">
      <header className="App-header">
        <img src={logo} className="App-logo" alt="logo" />
        <p>
          Edit <code>src/App.js</code> and save to reload.
        </p>
        <a
          className="App-link"
          href="https://reactjs.org"
          target="_blank"
          rel="noopener noreferrer"
        >
          Learn React
        </a>
      </header>
    </div>
  );
}

export default App;
~~~
App.js를 보시면 logo를 import해서 사용하는 것을 보실 수 있습니다. 이렇게 이미지, css를 import를 하면 경오레 해쉬갑이 할당되어 브로우저 캐싱을 효율적으로 사용할 수 있습니다.

### create-react-app의 명령어
- npm run start
먼저 start입니다. 개발환경에서 사용것으로 절대 배포에서는 사용하면안됩니다. 그것은 바로 최적화가 어있지 않기 때문입니다. 
- npm run bulid
bulid는 하면 정적파일이 생성되고 이것을 서버에는 생선된 빌드 파일을 배포하면 됩니다.
- npm run eject
eject는 react script를 사용하면 모든 설정을 추출하는 명령어로 create-react-app환경을 기반으로 환경을 설정하고 싶을때 사용하지만 관리측면에서 그냥 사용하는게 좋습니다.

### 마무리 
이렇게 오늘은 간단히 react가 무엇이고 왜 사용하는지 그리고 간단히 create-react-app을 통해 프로젝트를 생성해보았습니다. 여기서도 강의 적지않은 중요한 꿀팁들이 있었습니다. render함수는 순수함수로서 사용하고 img, css를 캐싱하기위해 import하는 등의 팁이
있었는데 공부하고 프로젝트를 진행해가면서 잊지않고 써먹을 수 있어야겠습니다.
